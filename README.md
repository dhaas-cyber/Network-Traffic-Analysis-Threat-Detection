# Threat Detection & Traffic Analysis

Objective

Analyze network traffic to identify normal baselines and abnormal behavior to detect potential indicators of compromise using packet inspection techniques.

Tools Used

- Wireshark

Methodology

- Captured network traffic using a packet analysis tool
- Filtered traffic based on IP addresses and ports
- Identified anomalies such as unusual connections or repeated requests
- Reviewed logs to trace activity patterns

Key Findings

- Identified unusual traffic patterns or suspicious connections
- Documented indicators of potential malicious activity

Skills Learned

- Stronger knowledge of port differentiation
- Network traffic analysis and packet inspection
- Log review and analysis
